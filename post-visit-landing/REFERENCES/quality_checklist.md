# Post-Visit 输出自检清单

生成完成后，逐条检查；任一关键项不满足，先修正再输出。

- [ ] 1. 是否先按 `transcript_quality` 分级，再决定输出深度
- [ ] 2. `完整清晰` 时，业务经验是否不少于 5 条
- [ ] 3. 每条业务经验是否都能引用具体原文，或明确标注来源
- [ ] 4. 场景收敛清单是否包含“Skill1 基准 / 现场新增 / Skill1 基准但现场否定”这些来源判断
- [ ] 5. 是否只对 `P0 / P1` 场景展开指标建设路径
- [ ] 6. 看板建议是否至少包含 1 个 Quick Win MVP
- [ ] 7. 推进建议表是否包含 `Phase 1 - Quick Win MVP`
- [ ] 8. 客户汇报材料是否避免技术词、内部判断、置信度和风险暴露
- [ ] 9. 若无 `skill1_output`，摘要里是否明确写“无 Pre-Visit 基准对照”
- [ ] 10. `碎片化` 时是否停止输出完整方案，只输出草稿和待确认清单；客户版不得泄露内部信息
