# 跨域破圈连接模式

## 核心连接链

```
销售预测偏差 → 供应链库存积压 → 财务现金流压力 → HR用工计划调整
```

这是帆软 CSM 的核心价值：发现部门间的数据/业务断点并串联起来。

## 连接模式详解

### 1. 销售 → 供应链
- **断点**：销售预测准确率低 → 备货要么积压要么缺货
- **识别词**："预测不准"、"备货不知道怎么备"、"多仓库存口径不一"
- **关联指标**：OTIF、Perfect Order Rate、Inventory Turnover、DIO

### 2. 供应链 → 财务
- **断点**：库存积压 → 资金占用 → Cash Conversion Cycle 恶化
- **识别词**："资金占用"、"现金周期"、"库存周转"
- **关联指标**：CCC、DIO、DSO、库存金额/呆滞率

### 3. 财务 → HR
- **断点**：现金流压力 → 用工计划调整（缩招/优化）
- **识别词**："人工成本"、"预算收紧"、"招聘冻结"
- **关联指标**：人工成本占比、人均营收、编制达成率

## 实际案例映射

| 客户场景 | 表面问题 | 背后断点 | 破圈机会 |
|---------|---------|---------|---------|
| 制造业30亿营收 | 库存账实不符 | 多系统数据不打通 | FDL整合供应链+财务 |
| 零售企业 | 月底盘点才发现积压 | 实时库存监控缺失 | BI看板+预警机制 |
| 某汽车零部件 | OEE仅45% | 异常停机响应慢 | MES+ERP+QMS整合 |

## 破圈提问框架

1. "你们销售预测偏差大概多少？"
2. "库存周转天数是多少？觉得合理吗？"
3. "现金流最近压力大吗？"
4. "今年用工计划有什么调整？"

顺着这4个问题，可以自然串联起跨域断点。
