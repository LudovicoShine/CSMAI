# Pre-Visit 输出自检清单

生成完成后，逐条检查；任一关键项不满足，先修正再输出。

- [ ] 1. 是否先做了信息完整性检查；若信息不足，是否先返回补齐问卷
- [ ] 2. 客户与部门判断是否明确标注“公开资料判断”或“基于行业推测”
- [ ] 3. 是否明确判断本次拜访属于业务场景挖掘、数据治理，或双线并行
- [ ] 4. 是否输出了 6 个主场景
- [ ] 5. 每个主场景是否满足 `scene_table_schema.md` 的 10 字段要求
- [ ] 6. 主场景是否贴合 `visit_context` 与 `visit_goal`，没有泛行业套话
- [ ] 7. 是否输出了完整现场引导问题，而不是只有方法论摘要
- [ ] 8. 钩子指标雷达是否引用了 `hook_keywords.md` 或等价原话信号
- [ ] 9. 推进建议是否明确写成“预测”，且与场景优先级一致
- [ ] 10. 场景地图是否同时包含 8.1、8.2、8.3 三层结构
