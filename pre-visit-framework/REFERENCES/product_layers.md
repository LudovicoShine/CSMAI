# 帆软产品分层判断标准

CSM 在拜访中要能根据客户描述的场景，快速判断该用哪一层产品。原则：**先匹配业务动作，再选产品**，不要让客户为产品而做场景。

---

## FineDataLink (FDL)｜数据集成 + 治理底座

**适用场景**：
- 多源数据接入（ERP / WMS / TMS / CRM / OA / 财务 / HR 系统同步）
- 离线 / 实时 / 增量同步、数据清洗、宽表构建
- 库存快照、订单快照、财务期间快照等中间层加工
- 字段映射、口径治理、主题层定义

**判断标准**：
> 如果客户把大量复杂 SQL、跨库关联、重计算放在 BI 或 FR 里 → 应优先前移到 FDL

**典型客户信号**：
- "BI 里数据集越来越多，跑得很慢"
- "同一指标在不同看板里数字不一样"
- "每次报表都要重新连库"
- "口径打架"

---

## FineBI｜自助分析 + 经营驾驶舱

**适用场景**：
- 经营驾驶舱、供应链分析、财务经营分析、HR 人效分析
- 指标趋势、同比、环比、累计、钻取、筛选、预警
- 业务自助探索、专题分析

**判断标准**：
> 如果 BI 里数据集过多、口径不信、刷新不稳定 → 判断是否数据准备层缺失（推 FDL 前置）
> 如果客户想做"老板看一眼就懂"的看板 → BI 主战场

**典型客户信号**：
- "想做一个经营驾驶舱"
- "希望能下钻看明细"
- "现在用 Excel 拼透视表太慢"

---

## FineReport (FR)｜中国式复杂报表 + 填报

**适用场景**：
- 中国式复杂报表、固定格式日报 / 周报 / 月报
- 填报采集、打印导出
- 财务报表、合并报表、监管报表

**判断标准**：
> 适合报表展示和格式化输出，**不应承接过重数据加工逻辑**
> 涉及"必须长成这个样子打印出来发出去"的场景 → FR

**典型客户信号**：
- "我们每月要给上级报这张表，格式必须一模一样"
- "需要填报采集后台数据"
- "要打印盖章"

---

## 简道云｜业务数据采集 + 轻流程 + 移动协同

**适用场景**：
- 业务数据采集（替代/补充 Excel 填报）
- 跨部门轻流程审批（请假、报销、工单、巡检）
- 移动端汇报与数据填报入口
- 业务侧的轻应用搭建（无需开发）

**判断标准**：
> 当客户痛点是"业务数据没有数字化入口"、"Excel 满天飞"、"流程靠微信群跑" → 简道云
> 与 FineReport 区别：FR 侧重展示与导出，简道云侧重**采集与流转**
> 与 FineBI 区别：简道云生产数据，FineBI 消费数据

**典型客户信号**：
- "现场数据还在用纸质表 / Excel"
- "审批流程都靠微信群"
- "想让业务部门自己录数据，不用每次找 IT"
- "需要移动端填报"

**协同模式**：
- 简道云采集 → FDL 同步入仓 → FineBI 分析 → FineReport 出报表

---

## Dora（Data Agent 平台）｜AI 数字员工

**适用场景**：
- 自然语言问数（让业务用户用对话方式查数据）
- 自动异常预警与归因推送
- 例行报告自动生成与定期推送
- 数据摘要、趋势解读、智能简报
- 跨系统查询的自主任务执行（数字员工）

**判断标准**：
> 当客户已经有 BI 但反馈"看的人少、用的人浅"、"老板没时间打开看板" → Dora 把数据"送上门"
> 当客户痛点是"问个数据要找 IT"、"每周都要花半天整理报告" → Dora 自动化

**典型客户信号**：
- "希望能用聊天的方式问数据"
- "异常发现总是滞后"
- "每周报告都要人肉整理"
- "高层不会用 BI，但需要他看到关键数据"

**协同模式**：
- FDL/FineBI 提供数据底座 → Dora 作为对话与自动化的"出口"
- FineBI 看板 + Dora 智能简报 = 双通道分发

## 产品组合决策速查

| 客户主诉 | 推荐组合 |
|---|---|
| 想做经营驾驶舱 | FineBI 主 + FDL 底座 |
| 报表打不开、跑得慢 | FDL 前置 + FineBI / FR 复用 |
| 每月报固定格式的表 | FineReport |
| 业务数据没数字化入口 | 简道云 + FDL 同步 |
| 现场移动填报 + 流程 | 简道云 |
| 老板不打开看板 / 想用聊天问数 | Dora + FineBI |
| 异常预警 / 自动报告 | Dora + FineBI |
| 跨部门数据打架 / 口径治理 | FDL（先治理）+ FineBI（再呈现）|
