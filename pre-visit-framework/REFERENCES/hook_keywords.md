# 钩子关键词词库

用于现场快速识别切入点。看到客户原话后，优先从左到右判断：是否命中关键词、应落到哪个场景、第一句追问问什么。

## 使用规则

- 一个原话可命中多个场景，但优先选离管理动作最近的那个
- 追问要问动作、频率、责任人，不要直接问“要不要做个看板”
- 若连续命中 2 个以上治理类关键词，说明要考虑“双线并行”

| 客户原话关键词 | 命中场景 | 建议追问 |
|---|---|---|
| 老板总追问 | 经营总览驾驶舱 | 老板最常问的三个问题分别是什么？ |
| 每次经营会都得先统一数字 | 经营总览驾驶舱 / 公共数据口径与字段治理 | 现在经营会上最容易打架的三个数是什么？ |
| 材料每次都要重新拼 | 经营材料自动化汇总场景 | 上次做这份材料花了多久？哪些部分每次都得重来？ |
| 口径打架 | 公共数据口径与字段治理场景 | 最后通常谁拍板？有没有成文口径？ |
| 项目状态经常说不清 | 项目推进与异常闭环看板 | 你们现在用什么标准定义项目是红灯还是黄灯？ |
| 问题总在群里追 | 跨部门协同与责任闭环场景 | 一个问题从暴露到闭环，中间通常经过哪些人？ |
| 问题发现太晚 | 经营异常预警与提前干预场景 | 这类问题通常是在什么时候、什么会上才被发现？ |
| BI 里也有，但没人看 | 场景诊断 / 价值实现层 | 是场景不对、数不信，还是没人知道什么时候该看？ |
| 一个指标不同地方不一样 | 公共数据口径与字段治理场景 | 哪几个系统都在算这个指标？差异主要出在哪？ |
| 导出以后还得再改 | 经营材料自动化汇总场景 | 导出后通常还要补哪些内容？格式还是口径问题？ |
| 老板没时间打开看板 | 一页式汇报 / Dora 智能简报 | 老板更习惯看一页纸、邮件，还是消息推送？ |
| 还是习惯用 Excel | 经营材料自动化汇总 / 简道云采集 | 你们离不开 Excel 的那一步，到底是在录数、加工还是汇报？ |
| 人不够用 | 项目资源占用与负荷平衡场景 | 哪些项目或岗位最容易抢人？ |
| 今年想先做个 Demo 看看 | Quick Win MVP | 哪个场景做出来后，最容易让大家马上用起来？ |
| 业务部门不配合 | 跨部门协同与责任闭环 / 推进建议 | 他们不配合通常卡在数据、口径，还是责任边界？ |
| 这个字段没人说得清 | 公共数据口径与字段治理场景 | 这个字段在哪些报表或看板里最常被用到？ |
| 周会就盯这几个项目 | 项目推进与异常闭环看板 | 盯项目时最先看的 3 个状态是什么？ |
| 重点客户得单独看 | 重点客户/重点项目专题经营分析 | 哪些客户会被单独拎出来？判断标准是什么？ |
| 汇报前最怕临时改数 | 经营材料自动化汇总 / 数据治理 | 临时改的是源数据、统计口径，还是展示口径？ |
| IT 太忙了，别搞太重 | Quick Win MVP / 隐性约束 | 如果不能依赖 IT 做大量开发，业务侧能先接受什么轻量方案？ |

## 高频组合信号

| 组合信号 | 判断 | 建议动作 |
|---|---|---|
| 老板总追问 + 每次经营会先统一数字 | 高层价值场景强，且治理基础弱 | 先推经营总览，再圈定 10 个核心字段治理 |
| 材料每次都要重新拼 + 导出以后还得再改 | 汇报自动化价值高 | 先做材料模板化 Quick Win |
| 口径打架 + 一个指标不同地方不一样 + 字段没人说得清 | 数据治理诉求明确 | 业务场景绑定治理，不单开治理项目 |
| 周会盯项目 + 项目状态说不清 + 问题总在群里追 | 运营闭环场景明确 | 先做项目推进与异常闭环 |
