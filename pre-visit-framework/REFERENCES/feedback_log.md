# 使用反馈记录

本文件用于沉淀 CSM、业务团队和平台使用过程中的反馈。每次发现输出质量问题、模型适配问题或业务表达问题，都追加到这里。后续更新 Skill 时，先读取本文件，再决定是否反写到 `SKILL.md`、schema 或其他 `REFERENCES/` 文件。

## 反馈使用规则

- 新反馈优先写成“问题现象 → 影响 → 建议改法 → 状态”
- 若反馈会稳定影响输出质量，应反写到 `SKILL.md` 或对应 schema
- 若反馈只是个案，先留在本文件，不急着改主规则
- 平台如果支持 agent 反写，可让 agent 先读本文件，再更新 Skill 自身
